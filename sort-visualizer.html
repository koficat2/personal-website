Bubble Sort
O(n²) avg/worst
Repeatedly swaps adjacent elements that are in the wrong order. Simple but slow for large inputs.
Selection Sort
O(n²) avg/worst
Finds the minimum element each pass and places it at the front. Minimizes swaps but not comparisons.
Insertion Sort
O(n²) avg · O(n) best
Builds a sorted subarray one element at a time. Fast on nearly-sorted data.
Merge Sort
O(n log n) always
Divides and conquers — splits the array, sorts each half, then merges. Stable and predictable.
